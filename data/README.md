# Data Layer — Cleaning Pipeline

Cleans the raw **NYC Yellow Taxi Trip** data (January 2019) and produces a
normalized, feature-enriched dataset for the database load. The final output
columns and types match the agreed schema contract (see `CLAUDE.md` §4).

## Pipeline stages

`pipeline.py` runs five stages in order, then exports:

1. **load** (`load.py`) — read the raw trips CSV with explicit dtypes (nullable
   integers where blanks exist) and parsed pickup/dropoff datetimes.
2. **integrate** (`integrate.py`) — join `taxi_zone_lookup.csv` for pickup /
   dropoff borough + zone, and derive each zone's centroid lat/lon from the
   `taxi_zones` shapefile (reprojected EPSG:2263 → 4326).
3. **clean** (`clean.py`) — handle missing values, drop exact duplicates, and
   remove invalid/outlier rows, attributing every removal to a rule.
4. **features** (`features.py`) — derive `trip_duration_min`, `avg_speed_mph`,
   `fare_per_mile`, `tip_pct`, `is_cross_borough`, `pickup_hour`,
   `pickup_day_of_week`, `time_of_day` (all divisions guarded against zero).
5. **normalize** (`features.py`) — rename to the snake_case contract, enforce
   final dtypes, and emit exactly the §4 columns in order.

## Setup

From the repository root:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip3 install -r data/requirements.txt
```

## Raw inputs

Place the raw data (git-ignored, not in the repo) under `data/raw/`:

```
data/raw/
  yellow_tripdata_2019-01.csv     ~657 MB, ~7.67M rows
  taxi_zone_lookup.csv            LocationID -> borough / zone
  taxi_zones/                     taxi_zones.shp (+ .dbf/.prj/.shx/...)
```

## Run

```bash
python3 -m data.src.pipeline
```

Single full-file pass (~7.67M rows, ~1 min, a few GB of RAM). Logs row counts at
each stage and where each output was written. Chunked reading is available via
`load_trips(chunksize=...)` if the file ever outgrows memory.

## Outputs

| Path | What | Committed? |
|---|---|---|
| `data/processed/trips_clean.parquet` | full cleaned dataset (§4 schema) | No (git-ignored) |
| `data/sample/trips_clean_sample.csv` | 1,000-row reproducible sample for the DB load | Yes |
| `data/sample/zones_reference.csv` | `location_id, borough, zone, service_zone, centroid_lat, centroid_lon` for the zones dimension | Yes |
| `data/exclusion_log.csv` | per-rule removal counts + TOTAL (graded deliverable) | Yes |

The full cleaned dataset is intentionally **not** committed (size); the parquet
is regenerated by re-running the pipeline. The sample, zones reference, and
exclusion log are the small artifacts handed to the database layer.
